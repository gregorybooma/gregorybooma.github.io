From fork to stand-alone Github repo while keeping history

Date Sun 22 October 2017 Tags repositories / forks / forked-from / clone / mirror / mirror-push

It seems like a person should simply be able to “unfork” their repo when they no longer want it to be a fork. GitHub doesn’t offer such a command, but there is a way. This will drop the “forked from” attribution (and wikis, etc.) from the repo, but it keeps the commit history.

1) Make a bare clone of the forked repo

git clone --bare git@github.com:gregorybooma/osq.git

2) Create a new empty repo on GitHub with no extras such as readme, license, etc.

3) cd to the directory of the bare clone, and mirror push to the newly-created repo (the directory will have “.git” as the end of its name — don’t confuse for a file)

cd osq.git
git push --mirror git@github.com:gregorybooma/mitsg-osq.git

4) cd out of the bare clone’s directory to the desired local location for the new repo, and clone it there

cd ..
git clone git@github.com:gregorybooma/mitsg-osq.git

5) Delete the local and GitHub versions of the originally-forked repo
