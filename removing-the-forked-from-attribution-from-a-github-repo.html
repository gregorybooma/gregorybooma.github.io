Removing the forked from attribution from a GitHub repo

Date Sat 21 October 2017 Tags repositories / forks / forked-from / clone / mirror / mirror-push

The title to this post is a bit misleading. One cannot simply remove the “forked from” attribution from a repo. This is because a fork can be thought of as a workspace of sorts for contribution to a larger project — i.e. it is still part of that larger project. When a person wants to disconnect the fork from the larger project they cannot simply edit the “forked from” attribution.

github: publish
#   cd $(OUTPUTDIR) && git add --all && ghp-import -m "publish site (make github)" -b $(GITHUB_PAGES_BRANCH) $(OUTPUTDIR) -p
    git -C $(OUTPUTDIR) add --all
    git -C $(OUTPUTDIR) commit -v -a -m 'Pelican :: make github'
    git -C $(OUTPUTDIR) push -v -u origin $(GITHUB_PAGES_BRANCH)

Posts are published by putting source markdown (or other accepted format) in the ‘articles’ directory and running the following commands in a terminal (must be in top level directory of site):

make publish
make github

This generates an html file of the post in the ‘output’ directory, and publishes to GitHub Pages. Edits are committed by adding a ‘modified’ line in markdown, then running

make regenerate
make publish
make github

A lot more customization/automation is possible; but for now I have a nice, straightforward website with blog.
